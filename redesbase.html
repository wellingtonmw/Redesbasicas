IPv4: O Endereço Clássico

O IPv4 é um endereço de 32 bits, representado por quatro números (octetos) de 0 a 255. Por décadas, foi o principal sistema de endereçamento da internet, mas seu limite de 4,3 bilhões de endereços está praticamente esgotado.

Cada octeto representa 8 bits. Passe o mouse sobre o endereço ao lado para ver a representação binária de cada parte e entender sua estrutura fundamental.

Máscara de Rede: Dividindo o Endereço

A máscara de rede é um filtro que separa um endereço IP em duas partes: a porção de **rede** (que identifica o "bairro" digital) e a porção de **host** (que identifica a "casa" específica nesse bairro).

Use a calculadora ao lado para ver como diferentes máscaras alteram a divisão entre rede e host para um mesmo IP. Isso é fundamental para organizar redes e controlar o tráfego.

Gateway Padrão e NAT

Esses dois conceitos trabalham juntos para conectar sua rede local à internet. O Gateway é a "porta de saída", e o NAT é o "tradutor" que permite que vários dispositivos compartilhem um único endereço público.

IPv6: O Futuro da Rede

O IPv6 é a solução para o esgotamento do IPv4. Com um endereço de 128 bits, ele oferece um número quase infinito de endereços (340 undecilhões). Seu formato usa oito grupos de quatro dígitos hexadecimais.

Uma de suas características é a capacidade de ser abreviado. Use o botão ao lado para ver como um endereço longo pode ser compactado, tornando-o mais fácil de gerenciar.

Diagnóstico de Rede: Ping e Traceroute

Ping e Traceroute são ferramentas essenciais que usam o protocolo ICMP (Internet Control Message Protocol) para testar e depurar problemas de rede. O ICMP não transporta dados de usuário, mas sim mensagens de controle e erro.

Ping (Teste de Conectividade)

Objetivo: Verificar se um host de destino está alcançável e medir o tempo de ida e volta (latência).

Como funciona: Envia uma mensagem ICMP Echo Request. O destino, se online, responde com uma ICMP Echo Reply.

Traceroute (Mapeamento de Rota)

Objetivo: Descobrir a rota (a sequência de roteadores) que um pacote percorre até o destino.

Como funciona: Envia pacotes com um TTL (Time-To-Live) crescente. Cada roteador no caminho descarta o pacote quando o TTL expira e envia de volta uma mensagem ICMP Time Exceeded, revelando sua identidade.
